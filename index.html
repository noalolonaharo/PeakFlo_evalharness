What a per-dimension judge sees

Four dimensions rose. One retreated.

Scoring each execution against the rubric instead of collapsing it to one number splits the average apart. The aggregate gain is real — and so is what it's hiding.

v1 baseline v2 improved v2 regressed
Regression flagged

Factual accuracy fell 21% (4.75 → 3.75) while everything else climbed. The agent learned to be warmer and to close commitments — and started quoting balances that are wrong or imprecise. In collections, telling a customer the wrong number is the single most expensive thing the agent can do: it triggers disputes, erodes trust, and corrupts the ledger the whole workflow depends on.

Where the regression lives

Two executions did the damage.

Per-transcript scores localize it. Both offenders are voice calls where the agent stated a balance that doesn't match the ledger — exactly the channel where a number, once spoken, can't be unsaid. Expand any row for the transcript and the judge's reasoning.

Can we trust the judge that drives optimization?

The judge is blind exactly where it broke.

If the agent self-optimizes against an LLM judge, the judge's blind spots become the agent's blind spots. Calibrating the judge against human scores on a sampled subset shows it agrees with people everywhere — except on factual accuracy, where it runs lenient.

Why this is the real finding

A wrong dollar figure is plausible — only the ledger knows it's wrong, and the judge isn't given the ledger. So a system that grades itself this way will happily trade accuracy for tone and report itself as improving. The "smarter than the last execution" claim can be confidently false.

What I'd ship next

Close the blind spot before trusting the optimizer.

01

Ground-truth accuracy check

Don't let an LLM guess whether a quoted figure is right. Extract every amount and date the agent states and diff it against the ledger record deterministically. Factual accuracy becomes pass/fail truth, not a plausibility score.

02

Per-decision audit log

Record which value the agent quoted, its source, and the judge's verdict for every execution — so a regression like this is traceable in one query instead of surfacing in a customer dispute weeks later.

Scope & honesty: this runs on hand-authored synthetic transcripts, not Peakflo's live system — I don't have access to it, and an applicant shouldn't pretend to.
